## Beyond Media Queries

For years, we used media queries tied to the viewport width. But components don't care about the viewport — they care about *their own available space*.

### The Problem

A card component in a sidebar needs different styling than the same card in a main content area. Media queries can't distinguish between these contexts.

### Container Queries to the Rescue

```css
.card-container {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
  }
}

@container card (max-width: 399px) {
  .card {
    display: flex;
    flex-direction: column;
  }
}
```

### Real-World Use Cases

1. **Dashboard widgets** — Cards that adapt from compact to expanded layouts
2. **Sidebar navigation** — Collapsing labels to icons based on available space
3. **Form layouts** — Switching between stacked and inline field arrangements

### Browser Support

Container queries are supported in all modern browsers as of 2024. There's no reason not to use them today.



### With Tailwind CSS

Tailwind v4 has first-class container query support:

```html
<div class="@container">
  <div class="flex flex-col @md:flex-row @lg:grid @lg:grid-cols-3">
    <!-- Content adapts to container width -->
  </div>
</div>
```

> Container queries are the biggest CSS improvement since Flexbox. Start using them today.
